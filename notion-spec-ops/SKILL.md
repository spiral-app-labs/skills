# notion-spec-ops

Use when drafting/updating Notion specs that guide execution.

## Spec Quality Bar
Each spec must include:
- clear objective
- scope and non-scope
- system/workflow design
- acceptance criteria
- rollout plan
- risks and mitigations
- explicit next actions/tasks

## Writing Rules
- replace shallow drafts instead of patching weak structure
- be specific (owners, timings, dependencies)
- optimize for cold-start execution by another agent

## Sync Pattern
1. update local markdown source if used
2. mirror to Notion page
3. ensure local + Notion are consistent revision

## Founder-Facing Output
- one concise update with page link
- thread/reply can include technical breakdown
