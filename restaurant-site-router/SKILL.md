# Restaurant Site Router

Use this after lead qualification and current-site audit. The router prevents beautiful but wrong redesigns.

Primary reference: `restaurant-website-system/research/site-router.md`.
Strategic companion: `restaurant-website-system/research/restaurant-website-strategic-principles.md`.

## Intake

Capture these before choosing:

- Cuisine
- Service type
- Price point
- Atmosphere and visual tone
- Reservation/order intensity
- Revenue model
- Day-part
- Destination vs neighborhood role
- Occasion
- Footprint and years in operation
- Photography tier
- Conversion priorities

## Routing Contract

Write a routing note with:

- Chosen template slug.
- 2-3 rejected alternatives and why.
- Modifiers: palette, typography, CTA, menu, gallery, proof, heritage, motion.
- Required sections.
- Avoided patterns.
- Register risk, if any.

Then update Mission Control build stage to `routing` and mark `template-route-locked` only after the route is locked.
