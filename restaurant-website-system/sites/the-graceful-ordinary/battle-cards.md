# The Graceful Ordinary — Owner Battle Cards

## Strongest one-sentence close

“We kept the Graceful Ordinary identity and Resy workflow intact, then made the first 30 seconds online feel as confident, specific, and special-occasion-ready as the restaurant itself.”

## Owner talking points

- **This is not a generic redesign.** It uses the `Roma` archetype because The Graceful Ordinary is chef-driven, seasonal, editorial, refined-rustic, and reservation-led.
- **We preserved what already works.** Resy stays the booking path, the official story stays the core, and the site continues to point to menu/about/contact rather than replacing operations.
- **We made buried proof visible.** Kane County, Chicago Tribune, FSR, Eater, Forbes, Daily Herald, and Google review themes move from “hidden credibility” to early trust-building.
- **We made the mobile path faster.** Reserve, menu, call, contact, and directions are easier to reach without flattening the premium room/story.
- **We made it safer to sell.** The local v2 removes unsupported AAA/TripAdvisor/composite claims, uses `4.6` and `593 reviews`, uses exact Resy URL, and avoids fake reviewer names.
- **We added a concierge that behaves like a helpful host.** It recommends real menu items, shows menu cards, captures birthday/table intent, and routes allergy/large-party/unknown details to the team.

## Likely objections

| Objection | Response |
| --- | --- |
| “Our current site is already nice.” | “Agreed, the current site has strong taste and real content. The pitch is not that it is broken, it is that proof, menu clarity, and the mobile reservation path can work harder in the first 30 seconds.” |
| “We do not want to lose the brand tone.” | “The redesign keeps the ‘There is grace in the ordinary’ thesis and refined-rustic language, then gives it a more direct editorial path.” |
| “Will this replace Resy?” | “No. Resy stays the source of truth for availability and booking. The website simply gets more guests to the right Resy flow faster.” |
| “The menu changes weekly.” | “Exactly, so the site uses source-derived examples and clear market/weekly-changing disclaimers instead of pretending the menu is fixed.” |
| “What about dietary/allergy questions in the AI?” | “The concierge does not confirm unsupported dietary specifics. It routes those to a phone call and avoids fake operational promises.” |
| “Why does the pitch mention hours uncertainty?” | “Because the official site currently has Saturday lunch/brunch wording variance. We should confirm it rather than accidentally publishing the wrong operational detail.” |
| “Is the public preview already updated?” | “Not yet. Local v2 is implemented and evidenced; the public Vercel preview still needs redeploy before founder-facing claims.” |
