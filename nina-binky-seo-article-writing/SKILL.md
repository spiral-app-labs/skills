# nina-binky-seo-article-writing

Use this when Nina is writing a public Binky article meant to rank, earn trust, and convert the right reader into the next action.

## What To Read First
1. the task brief
2. any supplied keyword cluster / search intent notes
3. the target CTA and internal links, if provided
4. `skills/nina-research-decomposition/SKILL.md` only if the brief is vague or under-specified

## Required Input Brief
Do not start until these are either provided or explicitly inferred in the draft header:
- **primary keyword/topic**
- **audience**: who this is for and what stage they are in
- **search intent**: informational, comparison, problem-aware, symptom-aware, etc.
- **target CTA**: app download, related article, email capture, waitlist, or soft product nudge
- **internal linking context**: what Binky pages or related articles should be linked
- **constraints**: word-count target, tone notes, claims to avoid, medical-safety notes

If something is missing, make the minimum safe assumption and state it at the top of the draft instead of stalling.

## Writing Standard
Binky SEO content should feel:
- calm
- practical
- emotionally intelligent
- lightly warm, never cheesy
- medically careful
- written for tired new parents, not content marketers

Avoid:
- fluffy intros
- fake authority
- absolute claims without support
- alarmist or guilt-heavy phrasing
- robotic keyword stuffing

## Workflow
1. **Lock the intent**
   - what is the reader actually trying to solve?
   - what would make them bounce if we miss it?
   - what level of depth matches the query?

2. **Shape the angle**
   - pick one clear promise for the article
   - make sure the article angle is useful even if the reader never downloads Binky
   - prefer practical framing over inspirational vagueness

3. **Design the structure around SERP reality**
   Build for what should rank, not for what sounds pretty.
   - answer the main query early
   - use scannable headings that map to likely follow-up questions
   - include comparison/decision sections when the query implies choice
   - include short checklists, examples, or scripts when they help the parent act

4. **Write for trust first, SEO second**
   - use the keyword naturally in title, intro, at least one subhead, and metadata
   - use semantically-related language instead of repeating the same phrase
   - every section should either clarify, reassure, or help the reader act

5. **Place links intentionally**
   - link only where the next click is genuinely helpful
   - prefer 2-5 internal links that extend the reader journey naturally
   - use descriptive anchor text, not “click here”

6. **Place the CTA with taste**
   - soft CTA in intro only if it feels natural
   - stronger CTA after value has already been delivered
   - CTA should match intent: informational article = soft helpful next step, not hard sell

7. **Source carefully**
   - use credible, current sources for medical/development claims
   - separate evidence from interpretation
   - if a claim is soft or observational, write it that way
   - when sources are weak or mixed, say so cleanly

## Default Output Structure
Every draft should include these sections in order.

### 1) draft header
- primary keyword
- search intent
- audience
- target CTA
- suggested internal links
- source notes / evidence confidence

### 2) SEO package
- **title tag**: ~50-60 characters
- **meta description**: ~140-160 characters
- **url slug**
- **h1**
- **suggested social/share headline**

### 3) article outline
Use a clean H2/H3 outline before the full draft so the structure is obvious.

### 4) full draft
Default pattern:
- intro that answers the question fast
- core explanation / framework
- actionable steps or examples
- edge cases / what to watch for
- when to get more help or talk to a professional, when relevant
- conclusion with next best step

### 5) internal links block
List:
- target page/article
- why it belongs
- suggested anchor text

### 6) CTA block
Give the exact CTA copy to use in-body or at article end.

### 7) sources block
List sources used or recommended follow-up sources if the draft is a first pass.

## Structure Heuristics By Intent
### informational query
- answer quickly
- teach simply
- offer a soft related-resource CTA

### symptom/problem query
- validate the experience
- separate normal vs concerning clearly
- include “when to ask for help” section
- keep tone steady, not scary

### comparison / alternatives query
- use tables or bullets when useful
- define tradeoffs honestly
- do not force Binky as the answer to everything

### how-to query
- lead with steps
- include common mistakes
- include a fast version for exhausted parents

## Source Handling Rules
- prefer primary or expert-reviewed sources when discussing health, development, sleep, feeding, recovery, or safety
- if you cannot verify a claim, either remove it or soften it clearly
- do not invent stats
- do not cite weak listicles as if they are evidence
- if a draft is waiting on source verification, label it clearly

## Internal Linking Rules
At minimum, think about links to:
- adjacent Binky articles on the same parent problem
- a broader hub/category page if one exists
- the app/product only when the next step truly fits the reader's need

Good anchor text examples:
- “newborn night-feeding checklist”
- “how to handle unequal mental load after baby”
- “daily couple check-ins for new parents”

## CTA Rules
The CTA should feel like the next helpful step, not a trapdoor into marketing.

Good CTA patterns:
- try a calmer daily check-in with Binky
- read the companion guide on [related topic]
- save this checklist for tonight

Bad CTA patterns:
- aggressive urgency
- product pitch before value
- generic “download now” with no context

## Final Quality Check
Before handing off, verify:
- the article clearly matches one search intent
- the opening answers the query fast
- headings are useful and skimmable
- the keyword appears naturally, not awkwardly
- sources support any serious claims
- internal links are specific and useful
- CTA fits the article intent
- tone sounds like Binky, not SEO sludge

## Handoff Format
Return one package with:
1. brief assumptions
2. SEO package
3. outline
4. full draft
5. internal links
6. CTA copy
7. source list + confidence note

If the brief is weak, do not ask ten questions. make the safest smart assumptions, state them, and keep moving.
